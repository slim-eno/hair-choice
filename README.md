理发店个性化推荐系统
项目概述
基于位置与社交媒体文本分析的理发店个性化推荐系统，专注于为用户提供精准、个性化的理发店推荐服务。该系统整合了地理位置信息和社交媒体数据，通过先进的自然语言处理技术，构建了多维度的推荐模型，实现了距离优先、好评优先及发型风格优先的分类推荐策略。

系统架构
系统采用四层架构设计，确保各模块职责清晰、耦合度低，便于维护和扩展：

数据采集层：使用Scrapy爬虫合规获取理发店POI数据和小红书帖子
数据处理层：清洗、预处理和存储数据
模型训练层：训练情感分析、NER和LDA主题模型
推荐服务层：实现多种推荐算法和API服务
核心功能
1. 地理位置服务
支持基于高德地图API的位置定位和周边搜索
实现距离计算和地理围栏功能
提供地图可视化展示
2. 社交媒体分析
合规爬取小红书理发店相关帖子
情感分析：分析帖子情感倾向，计算好评分数
发型实体识别：识别文本中的发型名称和技术名词
主题发现：使用LDA发现潜在的发型风格主题
3. 个性化推荐
距离优先推荐：基于地理位置的就近推荐
好评优先推荐：基于用户评价的高分推荐
发型风格优先推荐：根据用户指定的发型风格推荐
主题优先推荐：根据服务主题进行推荐
4. 前端交互
响应式设计，适配多种设备
直观的地图交互界面
丰富的发型风格展示
实时推荐结果展示

交互流程
用户授权位置或手动输入位置
系统在地图上显示用户位置和附近理发店
用户选择推荐类型（距离、好评、风格、主题）
系统根据选择生成个性化推荐
展示推荐结果和详细分析
提供相关的社交媒体内容推荐
后端API
推荐接口
POST /api/recommend
参数：

user_lat：用户纬度
user_lon：用户经度
recommendation_type：推荐类型（distance, review, hair_style, topic）
style_entity：发型实体（可选，用于hair_style推荐）
target_topic：目标主题ID（可选，用于topic推荐）
top_n：返回结果数量（默认10）
radius：搜索半径（默认5000米）
理发店列表接口
GET /api/salons
理发店详情接口
GET /api/salons/<salon_id>
主题列表接口
GET /api/topics
发型风格列表接口
GET /api/hair_styles
数据流程
数据采集：通过Scrapy爬虫获取理发店POI数据和小红书帖子
数据预处理：清洗、去重、标准化数据格式
特征提取：计算距离、情感分数、实体识别、主题建模
推荐计算：根据用户选择的推荐类型，应用相应的推荐算法
结果展示：通过前端界面展示推荐结果和分析报告

运行模块
使用 main.py 运行各个模块：

# 运行POI爬虫
python main.py --run poi_spider --cities 北京 上海 广州 深圳

# 运行小红书爬虫
python main.py --run xiaohongshu_spider

# 处理数据
python main.py --run process_data

# 训练模型
python main.py --run train_models

# 运行推荐服务
python main.py --run recommendation

# 启动Web应用
python main.py --run web_app
